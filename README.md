# 民宿房态交流站

一个功能完整的民宿管理和预订交流平台，支持房东和客人两种角色，实现房态管理、价格设置、保洁安排、聊天咨询、预订确认、取消退款等完整业务流程。

## ✨ 功能特性

### 🏠 房东端
- **房态日历管理**：可视化日历显示，支持拖拽选择日期范围批量设置房态（可订/已预订/保洁中/维护中）
- **价格管理**：批量设置指定日期范围的价格，周末自动加价20%
- **预订管理**：查看预订列表，确认待确认预订，取消预订（联动日历、保洁、退款）
- **保洁安排**：自动生成退房保洁，手动添加保洁任务，保洁状态流转管理
- **退款管理**：处理退款申请，支持状态流转（待处理 → 处理中 → 已完成/已拒绝）
- **消息中心**：实时回复客人咨询，在聊天中直接确认预订意向

### 👤 客人端
- **查看房源**：浏览房源信息和日历房态
- **联系房东**：聊天咨询房源问题
- **预订意向**：在聊天中选择入住日期、人数，发送预订意向
- **我的预订**：查看预订记录，取消预订

### 🔄 核心业务流程

**预订流程：**
```
客人聊天咨询 → 确认入住日期和人数 → 发送预订意向 → 房东确认 → 日历锁房 → 自动生成退房保洁
```

**取消流程（三方联动）：**
```
点击取消预订 → 日历立即空出 → 保洁自动取消 → 生成80%退款 → 退款待房东处理
```

## 🛠 技术栈

- **React 18** + **TypeScript** - 类型安全的前端框架
- **Vite** - 快速的开发构建工具
- **Tailwind CSS** - 原子化CSS框架
- **date-fns** - 日期处理库
- **lucide-react** - 图标库
- **useReducer + Context** - 全局状态管理

## 🎯 使用说明

1. **角色切换**：右上角可以切换「房东」和「客人」角色
2. **房源切换**：左侧边栏选择不同房源
3. **测试取消流程**：
   - 在房东端「预订管理」中取消一条预订
   - 观察日历对应日期是否变绿（可订）
   - 查看「保洁安排」中相关保洁是否变为「已取消」
   - 查看「退款管理」中是否生成了退款申请

## 📝 核心数据模型

- **Property** - 房源信息
- **Booking** - 预订记录
- **Cleaning** - 保洁安排
- **Refund** - 退款记录
- **Conversation** - 会话
- **ChatMessage** - 聊天消息
- **DatePrice** - 日期价格和房态
